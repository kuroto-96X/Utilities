# Deploy Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `.github/workflows/deploy.yml` を GitHub Pages 用から Cloudflare Pages + Wrangler 用に書き換え、`git push origin main` だけで `https://96xtools.dev` に自動デプロイされる状態にする。

**Architecture:** GitHub Actions が .NET 10 で `dotnet publish` し、Wrangler CLI が `publish/wwwroot` を Cloudflare Pages プロジェクト `utilities` へアップロードする。カスタムドメイン `96xtools.dev` は Cloudflare 側で設定済み。base href は `/` のまま変更不要。

**Tech Stack:** GitHub Actions, `actions/setup-dotnet@v4`, `cloudflare/wrangler-action@v3`, .NET 10 Blazor WASM

### Task 1: deploy.yml を Cloudflare Pages 用に書き換える

**Files:**
- Modify: `.github/workflows/deploy.yml`

- [ ] **Step 1: 既存の deploy.yml を確認する**

  現在の内容（参照のみ）:
  ```yaml
  name: Deploy to GitHub Pages
  on:
    push:
      branches: [ main ]
  permissions:
    contents: write
  jobs:
    deploy:
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v4
        - uses: actions/setup-dotnet@v4
          with:
            dotnet-version: '10.0.x'
        - name: Publish
          run: dotnet publish -c Release -o publish
        - name: Fix base href
          run: |
            sed -i 's|<base href="/"|<base href="/Utilities/"|g' publish/wwwroot/index.html
        - name: Add nojekyll
          run: touch publish/wwwroot/.nojekyll
        - name: Deploy
          uses: peaceiris/actions-gh-pages@v4
          with:
            github_token: ${{ secrets.GITHUB_TOKEN }}
            publish_dir: publish/wwwroot
  ```

- [ ] **Step 2: deploy.yml を以下の内容に全面書き換えする**

  `.github/workflows/deploy.yml` をこの内容に置き換える:
  ```yaml
  name: Deploy to Cloudflare Pages

  on:
    push:
      branches: [ main ]

  jobs:
    deploy:
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v4

        - uses: actions/setup-dotnet@v4
          with:
            dotnet-version: '10.0.x'

        - name: Publish
          run: dotnet publish -c Release -o publish

        - name: Deploy to Cloudflare Pages
          uses: cloudflare/wrangler-action@v3
          with:
            apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
            accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
            command: pages deploy publish/wwwroot --project-name=utilities
  ```

  **変更の意図:**
  - `Fix base href` 削除 → ルートドメイン運用なので `/` のまま正しい
  - `Add nojekyll` 削除 → GitHub Pages 専用、不要
  - `permissions: contents: write` 削除 → 不要
  - `peaceiris/actions-gh-pages` → `cloudflare/wrangler-action@v3`

- [ ] **Step 3: コミットして push する**

  ```bash
  git add .github/workflows/deploy.yml
  git commit -m "ci: deploy to Cloudflare Pages via Wrangler"
  git push origin main
  ```

### Task 2: GitHub Actions の実行を確認する

**Files:** なし（確認のみ）

- [ ] **Step 1: Actions の実行状況を確認する**

  `https://github.com/kuroto-96X/Utilities/actions` を開き、直前の push によってトリガーされたワークフローが表示されていることを確認する。

- [ ] **Step 2: ワークフローが成功することを確認する**

  ワークフローの実行をクリックし、全ステップが緑（✅）になるまで待つ（目安: 3〜5 分）。

  失敗した場合の主な原因と対処:
  | エラー内容 | 原因 | 対処 |
  |---|---|---|
  | `CLOUDFLARE_API_TOKEN` not found | GitHub Secret の名前が違う | Settings → Secrets で名前を確認 |
  | `project not found` | Cloudflare Pages のプロジェクト名が違う | ダッシュボードで名前を確認し、`command:` の `--project-name=` を合わせる |
  | dotnet build error | コードのビルドエラー | ローカルで `dotnet build` を実行して確認 |

- [ ] **Step 3: サイトが公開されていることを確認する**

  ブラウザで `https://96xtools.dev` を開き、Blazor アプリが表示されることを確認する。

  表示されない場合:
  - Cloudflare Pages ダッシュボード → `utilities` プロジェクト → **Custom domains** タブで `96xtools.dev` がアクティブ（緑）になっているか確認
  - DNS の浸透に数分かかることがある。少し待ってから再試行する
